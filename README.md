# 基于传统图像处理的车道线检测项目

## 1. 项目简介

本项目使用 Python 和 OpenCV 实现了一个基于传统图像处理方法的车道线检测系统。程序可以对道路图片和道路视频进行处理，通过透视变换、边缘检测、阈值分割、直方图定位、滑动窗口搜索和二次多项式拟合等步骤，实现左右车道线的检测与可视化。

该项目不依赖深度学习模型，主要用于理解自动驾驶感知中传统视觉车道线检测的基本流程。

## 4. 项目完成的主要工作

本项目完成了一个完整的传统车道线检测流程，主要包括以下工作。

---

### 4.1 透视变换参数计算

程序通过 `compute_perspective()` 函数，根据图像中的四个梯形点计算透视变换矩阵。

该步骤的作用是将原始摄像头视角下的道路区域转换为接近俯视视角的鸟瞰图，方便后续检测车道线。

主要功能包括：

1. 保存原始图像中的梯形感兴趣区域；
2. 构造透视变换后的矩形目标区域；
3. 计算原图到鸟瞰图的透视变换矩阵；
4. 计算鸟瞰图回到原图的反向透视变换矩阵；
5. 保存原图尺寸和透视变换后的图像尺寸。

核心函数：

```python
compute_perspective(width, height, pt1, pt2, pt3, pt4)
```

---

### 4.2 图像透视变换

程序通过 `warp()` 函数对图像进行透视变换。

该函数主要完成两件事：

1. 在原图上绘制透视变换前的梯形区域和变换后的目标矩形区域，方便调试；
2. 使用 `cv2.warpPerspective()` 将原图转换为鸟瞰图。

核心函数：

```python
warp(img, filename)
```

保存的中间结果包括：

| 前缀       | 含义             |
| -------- | -------------- |
| `persp_` | 带有透视变换区域标记的调试图 |
| `warp_`  | 透视变换后的鸟瞰图      |

---

### 4.3 边缘检测

程序通过 `edge_detection()` 函数进行边缘检测。

该函数使用 OpenCV 的 Scharr 算子计算图像在 x 方向上的梯度，用于提取车道线两侧的边缘信息。

核心代码：

```python
edge_x = cv2.Scharr(channel, cv2.CV_64F, 1, 0)
```

该方法可以突出图像中水平方向亮度变化明显的位置，因此适合检测接近竖直方向的车道线边缘。

核心函数：

```python
edge_detection(channel, filename)
```

保存结果前缀：

```text
edge_
```

---

### 4.4 阈值分割

程序通过 `threshold()` 函数生成车道线二值图。

该函数结合两类信息进行筛选：

1. 边缘强度信息；
2. 颜色或亮度通道信息。

具体过程如下：

1. 创建一张全黑二值图；
2. 根据图像高度设置动态边缘阈值；
3. 从上到下逐行判断边缘强度；
4. 将边缘强度大于阈值的像素设为白色；
5. 对亮度通道进行阈值筛选；
6. 将亮度在指定范围内的像素也设为白色；
7. 输出综合二值图和单独亮度阈值图。

核心函数：

```python
threshold(channel_threshold, channel_edge, filename)
```

保存的中间结果包括：

| 前缀                     | 含义            |
| ---------------------- | ------------- |
| `threshold_edge_only_` | 只基于边缘阈值得到的二值图 |
| `channel_only_`        | 原始亮度或颜色通道图    |
| `threshold_`           | 边缘和亮度综合后的二值图  |
| `threshold_other`      | 只基于亮度阈值得到的二值图 |

---

### 4.5 底部区域直方图统计

程序通过 `histogram()` 函数对二值图底部三分之一区域进行列方向求和。

该步骤的目的是寻找左右车道线在图像底部的大致起始位置。

核心代码：

```python
partial_img = img[img.shape[0] * 2 // 3:, :]
hist = np.sum(partial_img, axis=0)
```

直方图中峰值较高的位置通常对应车道线所在的横向位置。

核心函数：

```python
histogram(img, prefix, filename)
```

保存结果包括：

| 前缀           | 含义          |
| ------------ | ----------- |
| `hist_`      | 综合二值图的直方图   |
| `hist_solo_` | 单独亮度阈值图的直方图 |

---

### 4.6 左右车道线初始位置判断

程序定义了 `HistLanes` 类，用来保存通过直方图得到的左右车道线初始位置及其可信度。

核心类：

```python
class HistLanes:
    def __init__(self, x_left, x_right, left_confidence, right_confidence):
        self.x_left = x_left
        self.x_right = x_right
        self.left_confidence = left_confidence
        self.right_confidence = right_confidence
```

程序提供了两种车道线初始位置搜索方式：

1. `lanes_full_histogram()`：在整张直方图左右两半区域中寻找最大峰值；
2. `lanes_partial_histogram()`：根据历史检测结果，在上一帧附近的小范围内寻找峰值。

这样可以提高视频连续帧检测时的稳定性。

---

### 4.7 滑动窗口搜索车道线像素

程序通过 `slide_window()` 函数实现滑动窗口搜索。

该方法从图像底部开始，根据直方图得到的左右车道线起点，逐层向上搜索可能属于左右车道线的白色像素。

主要步骤包括：

1. 获取二值图中所有非零像素坐标；
2. 设置滑动窗口数量、窗口宽度和最小像素数量；
3. 从图像底部向上逐层建立左右窗口；
4. 搜索每个窗口内的白色像素；
5. 如果窗口内像素数量足够，则更新下一层窗口中心；
6. 保存左右车道线的像素索引；
7. 对搜索到的车道线像素进行拟合。

核心函数：

```python
slide_window(img, binary_warped, hist, num_windows, filename=None, prefix=None)
```

程序中使用的关键参数包括：

| 参数                 | 作用              |
| ------------------ | --------------- |
| `num_windows = 15` | 滑动窗口数量          |
| `margin = 80`      | 窗口左右搜索宽度        |
| `min_pixels = 25`  | 更新窗口中心所需的最少像素数量 |

保存结果前缀：

```text
window_
```

---

### 4.8 车道线数据封装

程序定义了 `Line` 类，用于保存单条车道线的信息。

核心内容包括：

```python
class Line:
    lane_indexes = None
    x = None
    y = None
    fit = None
    fitx = None

```

其中：

| 属性             | 含义              |

| `x`            | 车道线像素点的 x 坐标    |

程序还定义了 `SlideWindow` 类，用于同时保存左右车道线检测结果。

---

### 4.9 多项式拟合和平滑处理

程序通过 `fit_slide_window()` 函数对滑动窗口搜索到的左右车道线像素进行二次多项式拟合。

拟合形式为：

```text
x = a*y² + b*y + c
```

程序中使用：

```python
np.polynomial.polynomial.polyfit()
```

得到拟合参数。

为了减少视频检测中的抖动，程序使用了移动平均方法：

```python
moving_average(prev_average, new_value, beta)
```

其中 `beta = 0.92`，表示较大程度保留历史拟合结果，从而让车道线变化更加平滑。

---

### 4.10 车道线回投影与结果显示

程序通过 `show_lanes()` 函数将鸟瞰图中拟合得到的车道线重新投影回原始图像。

主要步骤包括：

1. 根据拟合曲线生成鸟瞰图中的车道线点；
2. 使用反向透视变换矩阵将点映射回原图；
3. 在原始图像上绘制左、右车道线；
4. 保存鸟瞰图和原图上的车道线检测结果。

核心函数：

```python
show_lanes(sw, img_warped, img_orig, filename)
```

保存结果包括：

| 前缀              | 含义            |
| --------------- | ------------- |
| `lanes_warped_` | 鸟瞰图中的车道线结果    |
| `lanes_orig_`   | 原始图像中的车道线检测结果 |

## 5. 整体算法流程

本项目完整流程如下：

```text
输入道路图像或视频帧
        ↓
透视变换，得到鸟瞰图
        ↓
转换到 HLS 颜色空间
        ↓
提取亮度通道
        ↓
Scharr 边缘检测
        ↓
边缘阈值 + 亮度阈值分割
        ↓
得到车道线二值图
        ↓
底部区域直方图统计
        ↓
确定左右车道线初始位置
        ↓
滑动窗口搜索左右车道线像素
        ↓
二次多项式拟合车道线
        ↓
移动平均平滑拟合结果
        ↓
将鸟瞰图车道线反投影回原图
        ↓
绘制并保存最终检测结果
        ↓
输出图片或视频
```

## 10. 注意事项

1. 运行前需要保证安装了 OpenCV、NumPy 和 Matplotlib；
2. 输入视频文件名需要与代码中的 `video-sd.mp4` 保持一致；
3. 测试图片需要放在 `test_images_sd` 文件夹中；
4. 透视变换的四个点需要根据具体视频画面调整；
5. 如果道路光照变化强烈、车道线模糊或存在遮挡，检测效果可能下降；
6. 本项目属于传统视觉算法，对复杂道路场景的鲁棒性有限。

## 12. 总结

本项目实现了一个完整的传统车道线检测系统。程序首先对道路图像进行透视变换，得到鸟瞰图；然后通过 Scharr 边缘检测和亮度阈值分割提取车道线候选区域；接着利用直方图确定左右车道线初始位置，并通过滑动窗口搜索车道线像素；最后进行二次多项式拟合，将检测结果反投影回原始图像，并输出最终车道线检测结果。

通过本项目，可以系统理解传统视觉车道线检测的基本原理和完整工程流程，为后续学习自动驾驶感知、目标检测、语义分割和深度学习车道线检测打下基础。
